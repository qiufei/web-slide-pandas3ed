Chapter 3: Built-In Data Structures, Functions, and Files

This chapter delves into the fundamental building blocks of Python that are essential for data analysis. We’ll explore Python’s built-in data structures, how to create reusable functions, and how to interact with files.

Note

While libraries like pandas and NumPy offer advanced functionalities for larger datasets, they are designed to work in conjunction with Python’s core data manipulation tools. Therefore, mastering these basics is extremely important.

Data Structures and Sequences

Python provides several versatile and powerful data structures. Let’s start with some of the most frequently used ones: tuples, lists, and dictionaries. Understanding these is a crucial step towards becoming proficient in Python.

Tuple: Definition

A tuple is a fixed-length, immutable sequence of Python objects. This means once a tuple is created, you cannot change its elements or its size.

Immutability

Immutability means that the contents of the tuple cannot be changed after it’s created. This provides data integrity. Think of it like a sealed container – you can see what’s inside, but you can’t swap things out.

Tuple: Creation

The simplest way to create a tuple is with a comma-separated sequence of values, usually enclosed in parentheses:

tup = (4, 5, 6)
print(tup)

Parentheses can often be omitted:

tup = 4, 5, 6
print(tup)

Tuple: Conversion and Accessing

You can convert any sequence or iterator to a tuple using the tuple() function:

my_list = [4, 0, 2]
my_tuple = tuple(my_list)
print(my_tuple)

string_tuple = tuple('string')
print(string_tuple)

Elements are accessed using square brackets [], and like most programming languages, Python uses zero-based indexing:

print(string_tuple[0])  # Access the first element

Tuple: Nested Tuples

Tuples can contain other tuples, creating nested structures:

nested_tup = (4, 5, 6), (7, 8)
print(nested_tup)
print(nested_tup[0]) # Access the first tuple
print(nested_tup[1][0]) #Access the first element from the second tuple.

Tuple: Immutability (Detailed Example)

While the objects within a tuple might be mutable (like a list), the tuple itself, once created, cannot be modified:

tup = tuple(['foo', [1, 2], True])
# tup[2] = False  # This will raise a TypeError

# However, if an element is mutable (like a list), you can modify it IN PLACE:
tup[1].append(3)
print(tup) # Output: ('foo', [1, 2, 3], True)
Caution

You cannot assign a new object to a slot in the tuple, but you can modify the contents of a mutable object within the tuple.

Tuple: Concatenation and Multiplication

Tuples can be concatenated using the + operator:

tuple1 = (4, None, 'foo')
tuple2 = (6, 0)
tuple3 = ('bar',)  # Note the comma for a single-element tuple
combined_tuple = tuple1 + tuple2 + tuple3
print(combined_tuple)

Multiplying a tuple by an integer repeats the tuple:

repeated_tuple = ('foo', 'bar') * 4
print(repeated_tuple)
Note

Only references to the objects are copied, not the objects themselves.

Tuple: Unpacking

Python allows you to “unpack” tuples, assigning the values to individual variables:

tup = (4, 5, 6)
a, b, c = tup
print(b)

Even nested tuples can be unpacked:

tup = 4, 5, (6, 7)
a, b, (c, d) = tup
print(d)

Tuple: Swapping Variables

A classic use of tuple unpacking is swapping variable values elegantly:

a, b = 1, 2
print(f"a: {a}, b: {b}")
b, a = a, b  # Swaps the values of a and b
print(f"a: {a}, b: {b}")

Tuple: *rest (and _)

Sometimes you want to “pluck” a few elements from the beginning of a tuple. The *rest syntax captures the remaining elements:

values = 1, 2, 3, 4, 5
a, b, *rest = values
print(a)
print(b)
print(rest)

By convention, the underscore _ is often used for unwanted variables during unpacking:

a, b, *_ = values # We don't care about the rest

Tuple: Methods (count)

Because tuples are immutable, they have very few built-in methods. A useful one is count():

a = (1, 2, 2, 2, 3, 4, 2)
count_of_2 = a.count(2)
print(count_of_2)  # Output: 4

count method returns the number of occurrences of a specified value.

List: Definition and Mutability

Lists, in contrast to tuples, are variable-length and mutable. You can change their contents and size after creation.

Mutability

Mutability means you can change the list’s elements, add new ones, or remove existing ones after the list has been created. This makes lists very flexible for storing and manipulating collections of data.

List: Creation

Lists are defined using square brackets [] or the list() type function:

a_list = [2, 3, 7, None]
tup = ('foo', 'bar', 'baz')
b_list = list(tup)  # Convert a tuple to a list
print(b_list)

You can modify elements in a list:

b_list[1] = 'peekaboo'
print(b_list)

List: Materializing Iterators

The list() function is frequently used to materialize an iterator or generator expression:

gen = range(10)
print(gen)  # This is a range object (an iterator)
my_list = list(gen)  # Convert the iterator to a list
print(my_list)

List: Adding and Removing Elements

  • append(): Adds an element to the end of the list.
b_list.append('dwarf')
print(b_list)
  • insert(): Inserts an element at a specific position.
b_list.insert(1, 'red') # insert 'red' at index 1
print(b_list)
Caution

insert is computationally more expensive than append because it has to shift subsequent elements.

List: Removing Elements (pop and remove)

  • pop(): Removes and returns an element at a specific index.
removed_element = b_list.pop(2)
print(removed_element)
print(b_list)
  • remove(): Removes the first occurrence of a specific value.
b_list.append('foo')  # Add another 'foo'
b_list.remove('foo')  # Removes the *first* 'foo'
print(b_list)

List: Checking for Membership (in and not in)

Use the in and not in keywords to check if a list contains a value:

print('dwarf' in b_list)
print('dwarf' not in b_list)
Note

Checking in or not in in list is slow. Checking membership in a list is much slower than with dictionaries or sets because Python has to perform a linear scan of the list, while it can check dictionaries and sets (which are based on hash tables) in constant time.

List: Concatenation and Combining

Similar to tuples, use + to concatenate lists:

list1 = [4, None, 'foo']
list2 = [7, 8, (2, 3)]
combined_list = list1 + list2
print(combined_list)

Use extend() to append multiple elements from another iterable:

x = [4, None, 'foo']
x.extend([7, 8, (2, 3)])
print(x)
Tip

extend() is generally faster than using + for combining lists, especially large ones, as + creates a new list and copies elements.

List: Sorting (sort and sorted)

  • sort(): Sorts the list in place (modifies the original list).
a = [7, 2, 5, 1, 3]
a.sort()
print(a)
  • key argument: provide customized sorting method.
b = ['saw', 'small', 'He', 'foxes', 'six']
b.sort(key=len)  # Sort by the length of the strings
print(b)

List: Slicing

Slicing allows you to select a section of a list using the start:stop notation within square brackets:

seq = [7, 2, 3, 7, 5, 6, 0, 1]
sub_list = seq[1:5]  # Elements from index 1 (inclusive) to 5 (exclusive)
print(sub_list)

seq[3:5] = [6,3] # replace elements using slicing
print(seq)

List: Slicing (Omitting Start/Stop and Negative Indices)

  • Omitting start: Defaults to the beginning of the list.
  • Omitting stop: Defaults to the end of the list.
  • Negative indices: Count from the end of the list.
print(seq[:5])  # First 5 elements
print(seq[3:])  # Elements from index 3 to the end
print(seq[-4:]) # Last 4 elements
print(seq[-6:-2]) # From 6th last (inclusive) to 2nd last (exclusive)

List: Slicing (Step)

A step can be used to select every nth element:

print(seq[::2])   # Every other element
print(seq[::-1])  # Reverse the list

List slicing illustration

0 1 2 3 4 5
H E L L O !
0 1 2 3 4 5
-6 -5 -4 -3 -2 -1 
string = "HELLO!"
print(string[2:4]) # slicing with positive indices
print(string[-5:-2]) # slicing with negative indices

Dictionary (dict): Definition and Key-Value Pairs

A dictionary (or dict) is a highly important built-in data structure. Also known as a hash map or associative array, it stores a collection of key-value pairs.

Key-Value Pairs

Each key is associated with a value. Keys must be unique and immutable (like strings, numbers, or tuples), while values can be of any data type.

Dictionary: Creation

Use curly braces {} and colons : to separate keys and values:

empty_dict = {}
d1 = {'a': 'some value', 'b': [1, 2, 3, 4]}
print(d1)

Dictionary: Accessing, Inserting, and Setting Elements

Access, insert, or set elements using the same syntax as lists, but with keys instead of indices:

d1[7] = 'an integer'  # Add a new key-value pair
print(d1)
print(d1['b'])       # Access the value associated with key 'b'

Dictionary: Checking for Keys (in)

print('b' in d1)  # Checks if 'b' is a *key* in d1

Dictionary: Deleting Values (del and pop)

  • del: Deletes a key-value pair.
d1[5] = 'some value'
d1['dummy'] = 'another value'
print(d1)
del d1[5]
print(d1)
  • pop(): Removes a key-value pair and returns the value.
ret = d1.pop('dummy')
print(ret)
print(d1)

Dictionary: keys() and values() Methods

  • keys(): Returns an iterator of the dictionary’s keys.
  • values(): Returns an iterator of the dictionary’s values.
  • items(): Returns an iterator of the dictionary’s items(key-value pairs).
print(list(d1.keys()))   # Convert the iterator to a list
print(list(d1.values())) # Convert the iterator to a list
print(list(d1.items()))
Note

The order of keys depends on the insertion order. The keys and values methods return iterators in the same order.

Dictionary: Merging with update()

The update() method merges one dictionary into another:

d1.update({'b': 'foo', 'c': 12})  # Updates existing key 'b', adds new key 'c'
print(d1)
Caution

update() changes dictionaries in place. Any existing keys in the dictionary passed to update() will have their old values discarded.

Dictionary: Creating from Sequences

# Not how you would normally do it, but illustrates the concept
key_list = ['a', 'b', 'c']
value_list = [1, 2, 3]
mapping = {}
for key, value in zip(key_list, value_list):
    mapping[key] = value
print(mapping)

# A more concise way using dict() and zip()
mapping = dict(zip(range(5), reversed(range(5))))
print(mapping)

zip function can pair up elements of multiple sequences. dict accepts a list of 2-tuples.

Dictionary: Default Values (get and setdefault)

# Common but verbose way to handle missing keys
if 'some_key' in some_dict:
    value = some_dict['some_key']
else:
    value = default_value

# More concise using get()
value = some_dict.get('some_key', default_value)

get returns None if the key is not present (or a specified default value), pop will raise an exception.

# setdefault for setting values conditionally
words = ['apple', 'bat', 'bar', 'atom', 'book']
by_letter = {}
for word in words:
    letter = word[0]
    by_letter.setdefault(letter, []).append(word)  # Use setdefault
print(by_letter)

setdefault(key, default): If key exists, return its value. If not, insert key with a value of default and return default.

Dictionary: defaultdict

The collections module provides defaultdict, which simplifies the creation of dictionaries where each value is initialized with a default value:

from collections import defaultdict

by_letter = defaultdict(list) # Use defaultdict for a more concise way
for word in words:
    by_letter[word[0]].append(word)
print(by_letter)

Dictionary: Valid Key Types (Hashability)

  • Keys must be immutable objects (like scalar types or tuples).
  • The technical term is hashability.
  • Check if an object is hashable using hash():
print(hash('string'))
print(hash((1, 2, (2, 3))))
# print(hash((1, 2, [2, 3])))  # TypeError: unhashable type: 'list'

# To use a list as a key, convert it to a tuple:
d = {}
d[tuple([1, 2, 3])] = 5
print(d)

Set: Definition and Uniqueness

A set is an unordered collection of unique elements. Think of them like dictionaries, but with only keys and no values.

Set: Creation

Create sets using curly braces {} (like dictionaries, but without colons) or the set() function:

set1 = {2, 2, 2, 1, 3, 3}  # Using curly braces
print(set1) # Output: {1, 2, 3}  (duplicates are removed)
set2 = set([2, 2, 2, 1, 3, 3])  # Using the set() function
print(set2)

Set: Operations (Union, Intersection, etc.)

Sets support mathematical set operations:

a = {1, 2, 3, 4, 5}
b = {3, 4, 5, 6, 7, 8}

# Union (| or union())
print(a.union(b))
print(a | b)

# Intersection (& or intersection())
print(a.intersection(b))
print(a & b)

# Difference (- or difference())
print(a.difference(b)) # elements in a but not in b
print(a - b)

Set operations

Function Alternative Syntax Description
a.add(x) N/A Add element x to set a
a.clear() N/A Reset set a to an empty state, discarding all of its elements
a.remove(x) N/A Remove element x from set a
a.pop() N/A Remove an arbitrary element from set a, raising KeyError if the set is empty
a.union(b) a \| b All of the unique elements in a and b
a.update(b) a \|= b Set the contents of a to be the union of the elements in a and b
a.intersection(b) a & b All of the elements in both a and b
a.intersection_update(b) a &= b Set the contents of a to be the intersection of the elements in a and b
a.difference(b) a - b The elements in a that are not in b
a.difference_update(b) a -= b Set a to the elements in a that are not in b
a.symmetric_difference(b) a ^ b All of the elements in either a or b but not both
a.symmetric_difference_update(b) a ^= b Set a to contain the elements in either a or b but not both
a.issubset(b) <= True if the elements of a are all contained in b
a.issuperset(b) >= True if the elements of b are all contained in a
a.isdisjoint(b) N/A True if a and b have no elements in common

Set: In-Place Operations

All set operations have in-place counterparts (e.g., a |= b for union):

c = a.copy()
c |= b  # In-place union (modifies c)
print(c)

d = a.copy()
d &= b
print(d)
Tip

In-place operation is more efficient, especially when handling large sets.

Set: Element Hashability

Like dictionary keys, set elements must be immutable and hashable.

my_data = [1, 2, 3, 4]
# my_set = {my_data}  # TypeError: unhashable type: 'list'
my_set = {tuple(my_data)}  # Convert the list to a tuple first
print(my_set)

Set: Subset and Superset

a_set = {1, 2, 3, 4, 5}
print({1, 2, 3}.issubset(a_set))   # Check if {1, 2, 3} is a subset of a_set
print(a_set.issuperset({1, 2, 3}))  # Check if a_set is a superset of {1, 2, 3}

Set: Equality

Sets are equal if and only if their contents are equal:

print({1, 2, 3} == {3, 2, 1})

Built-In Sequence Functions: enumerate

enumerate is a useful function when you need to keep track of the index while iterating over a sequence:

# Instead of doing this:
collection = ['foo', 'bar', 'baz']
i = 0
for value in collection:
    # do something with value
    print(f"Index: {i}, Value: {value}")
    i += 1

# Use enumerate:
for i, value in enumerate(collection):
    print(f"Index: {i}, Value: {value}")
    # do something with value

enumerate returns a sequence of (index, value) tuples.

Built-In Sequence Functions: sorted

The sorted function returns a new sorted list from any sequence:

print(sorted([7, 1, 2, 6, 0, 3, 2]))
print(sorted('horse race')) # Sorts the characters in the string
Note

sorted() returns a new list, while the sort() method of lists sorts in place.

Built-In Sequence Functions: zip

zip “pairs up” elements from multiple sequences, creating a list of tuples:

seq1 = ['foo', 'bar', 'baz']
seq2 = ['one', 'two', 'three']
zipped = zip(seq1, seq2)  # Create a zip object
print(list(zipped))     # Convert the zip object to a list of tuples

zip can take any number of sequences. The number of elements produced is determined by the shortest sequence:

seq3 = [False, True]
print(list(zip(seq1, seq2, seq3)))

Built-In Sequence Functions: zip (with enumerate)

A common use of zip is to iterate over multiple sequences simultaneously, often combined with enumerate:

for i, (a, b) in enumerate(zip(seq1, seq2)):
    print(f'{i}: {a}, {b}')

Built-In Sequence Functions: reversed

reversed iterates over a sequence in reverse order:

print(list(reversed(range(10))))  # reversed() returns a generator
Note

reversed is a generator, so it doesn’t create the reversed sequence until materialized (e.g., with list or a for loop).

List, Set, and Dictionary Comprehensions

Comprehensions provide a concise way to create new lists, sets, or dictionaries by applying an expression and an optional filter to an existing sequence.

List Comprehension: Basic Form

# General form: [expr for val in collection if condition]

# Equivalent for loop:
result = []
for val in collection:
    if condition:
        result.append(expr)

Example:

strings = ['a', 'as', 'bat', 'car', 'dove', 'python']
upper_case_long_strings = [x.upper() for x in strings if len(x) > 2]
print(upper_case_long_strings)

Set Comprehension

# General form: {expr for val in collection if condition}
strings = ['a', 'as', 'bat', 'car', 'dove', 'python']
unique_lengths = {len(x) for x in strings}  # Create a set of unique string lengths
print(unique_lengths)

# using map function
print(set(map(len, strings)))

Set comprehension is very similar to list comprehensions, but it uses curly braces {} and creates a set (unique elements).

Dictionary Comprehension

# General form: {key_expr: value_expr for val in collection if condition}

# Example: Create a dictionary mapping strings to their locations in a list
strings = ['a', 'as', 'bat', 'car', 'dove', 'python']
loc_mapping = {val: index for index, val in enumerate(strings)}
print(loc_mapping)

Nested List Comprehensions

List comprehensions can be nested:

all_data = [['John', 'Emily', 'Michael', 'Mary', 'Steven'],
            ['Maria', 'Juan', 'Javier', 'Natalia', 'Pilar']]

# Find names with two or more 'a's
names_of_interest = [name for names in all_data for name in names
                     if name.count('a') >= 2]
print(names_of_interest)


# Flatten a list of tuples into a single list
some_tuples = [(1, 2, 3), (4, 5, 6), (7, 8, 9)]
flattened = [x for tup in some_tuples for x in tup]
print(flattened)

# List comprehension inside a list comprehension.
flattened = [[x for x in tup] for tup in some_tuples]
print(flattened)
Note

The order of the for expressions in a nested list comprehension is the same as it would be in nested for loops.

3.2 Functions

Functions are a fundamental way to organize and reuse code in Python. They enhance readability and maintainability.

Function Declaration (def)

Functions are declared using the def keyword:

def my_function(x, y):
    """This is a docstring, explaining what the function does."""
    return x + y

result = my_function(1, 2)
print(result)
print(my_function.__doc__)
  • The return statement sends a value back to the caller.
  • If a function reaches the end without a return statement, it implicitly returns None.

Function Arguments (Positional and Keyword)

def my_function2(x, y, z=1.5):  # z is a keyword argument with a default value
    if z > 1:
        return z * (x + y)
    else:
        return z / (x + y)

print(my_function2(5, 6, z=0.7))  # Using the keyword
print(my_function2(3.14, 7, 3.5))  # Positional arguments
print(my_function2(10, 20))        # Using the default value for z
  • Positional arguments must be provided in the correct order.
  • Keyword arguments can be specified in any order and often have default values.
  • Keyword arguments must follow positional arguments.

Namespaces, Scope, and Local Functions

  • Variables assigned within a function are in the local namespace by default.
  • The local namespace is created when the function is called and destroyed when the function finishes.
def func():
    a = []  # 'a' is local to the function
    for i in range(5):
        a.append(i)

func()
# print(a)  # NameError: name 'a' is not defined (outside the function's scope)
  • You can access variables in enclosing scopes (e.g., global variables), but to modify them, you generally need to use the global or nonlocal keywords.
a = []  # 'a' is a global variable

def func2():
    for i in range(5):
        a.append(i)  # Modifies the *global* 'a'

func2()
print(a)

b = None # 'b' is a global variable
def bind_b_variable():
    global b  # Declare 'b' as global to modify it within the function
    b = []
bind_b_variable()
print(b)
Caution

Generally, avoid excessive use of the global keyword. It’s often better to design your code to minimize reliance on global state for better modularity.

Returning Multiple Values

Functions can return multiple values using tuples (or other data structures):

def f():
    a = 5
    b = 6
    c = 7
    return a, b, c  # Returns a tuple

x, y, z = f()  # Unpack the returned tuple
print(x, y, z)

# Or return a dictionary
def f2():
  a = 5
  b = 6
  c = 7
  return {'a' : a, 'b' : b, 'c' : c}

result = f2()
print(result)

Functions Are Objects

Functions in Python are first-class objects, which means you can:

  • Pass them as arguments to other functions.
  • Assign them to variables.
  • Return them from other functions.
# Example: Data cleaning
import re # regular expression operations

states = ['   Alabama ', 'Georgia!', 'Georgia', 'georgia', 'FlOrIda',
          'south   carolina##', 'West virginia?']

def clean_strings(strings):
    result = []
    for value in strings:
        value = value.strip()  # Remove whitespace
        value = re.sub('[!#?]', '', value)  # Remove punctuation
        value = value.title()  # Title case
        result.append(value)
    return result

print(clean_strings(states))

# Alternative approach using a list of functions:
def remove_punctuation(value):
    return re.sub('[!#?]', '', value)

clean_ops = [str.strip, remove_punctuation, str.title]

def clean_strings_functional(strings, ops):
    result = []
    for value in strings:
        for function in ops:
            value = function(value)
        result.append(value)
    return result

print(clean_strings_functional(states, clean_ops))


# Using functions with map:
for x in map(remove_punctuation, states):
  print(x)

This functional approach makes the code very flexible and reusable.

Anonymous (Lambda) Functions

Lambda functions are small, anonymous functions defined using the lambda keyword:

# Equivalent to:
# def short_function(x):
#     return x * 2

equiv_anon = lambda x: x * 2
print(equiv_anon(4))

Lambda functions are particularly useful when passing short, simple functions as arguments to other functions:

def apply_to_list(some_list, f):
    return [f(x) for x in some_list]

ints = [4, 0, 1, 5, 6]
result = apply_to_list(ints, lambda x: x * 2) # Pass a lambda function
print(result)

# Another example
strings = ['foo', 'card', 'bar', 'aaaa', 'abab']
strings.sort(key=lambda x: len(set(x))) # Sort by number of distinct letters
print(strings)

Generators

Generators are a special kind of iterator that produce values on demand, rather than computing them all at once. This makes them very memory-efficient, especially when working with large sequences.

def squares(n=10):
    print('Generating squares from 1 to %d' % n ** 2)
    for i in range(1, n + 1):
        yield i ** 2  # Use 'yield' instead of 'return'

gen = squares()  # Create a generator object
print(gen)
# Nothing is computed until you request values:
for x in gen:
    print(x, end=' ')
  • Use the yield keyword instead of return.
  • Generators don’t execute immediately. They produce values only when iterated over.

Generator Expressions

Generator expressions are concise ways to create generators, analogous to list/set/dict comprehensions:

# List comprehension: [x ** 2 for x in range(100)]
# Generator expression:
gen = (x ** 2 for x in range(100)) # using parenthese
print(gen)
print(sum(gen))

# Can be used as function arguments:
print(sum(x ** 2 for x in range(100)))
print(dict((i, i **2) for i in range(5)))

itertools Module

The itertools module provides a collection of useful generators for common data algorithms:

import itertools

def first_letter(x):
    return x[0]

names = ['Alan', 'Adam', 'Wes', 'Will', 'Albert', 'Steven']

for letter, names_iter in itertools.groupby(names, first_letter):
    print(letter, list(names_iter)) # names is a generator

itertools.groupby groups consecutive elements based on the return value of a function.

Useful itertools Functions

Function Description
chain(*iterables) Generates a sequence by chaining iterators.
combinations(iterable, k) Generates k-tuples of elements, ignoring order, without replacement.
permutations(iterable, k) Generates k-tuples of elements, respecting order.
groupby(iterable[, keyfunc]) Generates (key, sub-iterator) for each unique key.
product(*iterables, repeat=1) Generates the Cartesian product of input iterables (similar to nested for loops).
Note

Check out the official Python documentation for the itertools module. It’s a treasure trove of useful tools!

Errors and Exception Handling

Handling errors (exceptions) gracefully is crucial for writing robust code.

try-except Blocks

def attempt_float(x):
    try:
        return float(x)
    except:  # Catches *any* exception
        return x

print(attempt_float('1.2345'))
print(attempt_float('something'))
  • The try block contains the code that might raise an exception.
  • The except block is executed if an exception occurs in the try block.

Catching Specific Exceptions

def attempt_float2(x):
    try:
        return float(x)
    except ValueError:  # Only catch ValueError
        return x

# print(attempt_float2((1, 2)))  # TypeError will not be caught

You can specify the type of exception to catch. This is generally preferred over catching all exceptions, as it allows you to handle specific errors appropriately.

Catching Multiple Exception Types

def attempt_float3(x):
    try:
        return float(x)
    except (TypeError, ValueError):  # Catch multiple exception types
        return x

Use a tuple of exception types to catch multiple kinds of exceptions.

finally and else

  • finally: Code in the finally block always executes, regardless of whether an exception occurred or not. Useful for cleanup (e.g., closing files).

  • else: Code in the else block executes only if the try block completed without raising an exception.

# Example illustrating finally
f = open("tempfile.txt", mode = "w")
try:
    # write_to_file(f)  # Assume this function might raise an exception
    f.write("example text")
finally:
    f.close()  # Always close the file
    import os # operating system operation
    os.remove("tempfile.txt") # Removes the file named "tempfile.txt"
# Example illustrating else
f = open("tempfile.txt", mode = "w")
try:
    #write_to_file(f)
    f.write("example text")
except:
    print('Failed')
else:
    print('Succeeded')
finally:
    f.close()
    import os
    os.remove("tempfile.txt")

Exceptions in IPython

IPython provides helpful stack traces (tracebacks) when exceptions occur, making debugging easier. You can control the level of detail using the %xmode magic command (Plain, Context, Verbose).

3.3 Files and the Operating System

Interacting with files is a common task in data analysis.

Opening Files (open)

path = 'examples/segismundo.txt'  # Replace with your file path
# Use a raw string (r'...') or escape backslashes for Windows paths
# path = r'C:\Users\YourName\Documents\myfile.txt'

# default encoding varies from platform to platform, better to specify explicitly.
f = open(path, encoding='utf-8')  # Open in read mode ('r' is the default)
#f = open(path, mode = "r", encoding='utf-8') # is equal to previous line
for line in f:
    print(line.rstrip()) # remove trailing whitespaces.

f.close() # close the file after using it.
  • open(path, mode='r', encoding=None) opens a file.
    • path: The file path (relative or absolute).
    • mode: ‘r’ (read), ‘w’ (write), ‘a’ (append), ‘x’ (create, fail if exists), ‘rb’ (binary read), ‘wb’ (binary write), etc.
    • encoding: Specify the file encoding (e.g., ‘utf-8’ for text files with Unicode characters). It is a good habit to set encoding parameter.
  • It’s essential to close files after you’re finished with them using f.close().

Reading from Files

# reading all the lines
f = open(path, encoding="utf-8")
lines = [x.rstrip() for x in f]  # Read lines, remove trailing whitespace
print(lines)
f.close()

Using with for Automatic File Closing

The with statement provides a cleaner way to handle files, automatically closing them when the block finishes:

with open(path, encoding="utf-8") as f:
    lines = [x.rstrip() for x in f]
print(lines)
# The file 'f' is automatically closed here
Tip

Using with is the recommended way to work with files in Python. It ensures proper cleanup, even if errors occur.

File Modes

Mode Description
r Read-only mode
w Write-only mode; creates a new file (erasing any existing file with the same name)
x Write-only mode; creates a new file, but fails if the file path already exists
a Append to existing file (creates the file if it doesn’t exist)
r+ Read and write
b Add to mode for binary files (e.g., ‘rb’ or ‘wb’)
t Text mode (automatically decoding bytes to Unicode); this is the default

read, seek, and tell

f1 = open(path, encoding = "utf-8")
print(f1.read(10))  # Read the first 10 *characters*

f2 = open(path, mode='rb')  # Open in binary mode
print(f2.read(10))  # Read the first 10 *bytes*

print(f1.tell())  # Current file position (characters)
print(f2.tell())  # Current file position (bytes)

import sys
print(sys.getdefaultencoding()) # get the default encoding

f1.seek(3)  # Move to the 3rd byte (character) in the file
print(f1.read(1))

f1.close()
f2.close()
  • read(n): Reads n characters (text mode) or bytes (binary mode).
  • tell(): Returns the current file position.
  • seek(position): Moves the file pointer to a specific position (in bytes).
  • Be cautious using seek with text files encoded in UTF-8, as seeking to a position in the middle of a multi-byte character can lead to errors.

Writing to Files

with open('tmp.txt', 'w', encoding = "utf-8") as handle:
    handle.writelines(x for x in open(path, encoding = "utf-8") if len(x) > 1) # write lines into a file

with open('tmp.txt', encoding = "utf-8") as f:
    lines = f.readlines()
print(lines)

import os
os.remove("tmp.txt")
  • write(string): Writes a string to the file.
  • writelines(list_of_strings): Writes a list of strings to the file.

Bytes and Unicode with Files

  • Default behavior is text mode (working with strings/Unicode).
  • Use mode='rb' or 'wb' for binary mode (working with raw bytes).
with open(path, encoding = "utf-8") as f:
    chars = f.read(10)  # read 10 characters
print(len(chars))

with open(path, 'rb') as f:
    data = f.read(10) # read 10 bytes
print(data)

print(data.decode('utf-8'))  # Decode bytes to a string

# print(data[:4].decode('utf-8')) # UnicodeDecodeError if incomplete character
  • UTF-8 is a variable-length encoding. Reading a specific number of characters might involve reading a different number of bytes.
  • In binary mode, read returns exactly the requested number of bytes.
  • When decoding bytes to strings, ensure that you’re decoding complete characters.
sink_path = 'sink.txt'
with open(path, encoding = "utf-8") as source:
    with open(sink_path, 'x', encoding='iso-8859-1') as sink:
        sink.write(source.read())

with open(sink_path, encoding='iso-8859-1') as f:
    print(f.read(10))

import os
os.remove(sink_path)

# caution
f = open(path, encoding = "utf-8")
print(f.read(5))
f.seek(4)
# print(f.read(1)) # UnicodeDecodeError: 'utf-8' codec can't decode byte 0xb1 in position 0: invalid start byte
f.close()
Caution

Beware using seek when opening files in any mode other than binary.
