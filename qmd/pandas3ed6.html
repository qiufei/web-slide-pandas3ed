Python for Data Analysis

Chapter 6: Data Loading, Storage, and File Formats

Introduction to Data Loading

  • Data loading is the crucial first step in any data analysis project. It involves reading data from various sources and making it accessible for processing.

  • The term parsing is often used interchangeably with data loading, referring to the process of interpreting text data and converting it into structured formats like tables with specific data types.

  • While various libraries exist, this chapter focuses on using pandas, a powerful Python library, for data input and output.

  • This chapter focuses on loading from text files, efficient storage format, database, and interact with web APIs.

Why Data Loading Matters

  • Foundation for Analysis: Without loading data, you can‚Äôt perform any analysis! ü§Ø
  • Data Variety: Data comes in many shapes and sizes: text files, databases, web APIs, and more. Learning to handle these different formats is essential.
  • Efficiency: Efficient data loading techniques can save you significant time and computational resources, especially with large datasets.
  • Data Cleaning: The loading process often involves initial data cleaning, such as handling missing values or inconsistent formatting.

Core Concepts: Data Mining and Machine Learning

Before diving into data loading, let‚Äôs clarify some fundamental concepts.

  • Data Mining: The process of discovering patterns, anomalies, and insights from large datasets. It‚Äôs like searching for gold ü™ô in a mountain of data!

  • Machine Learning (ML): A subfield of artificial intelligence (AI) where algorithms learn from data without explicit programming. Think of it as teaching a computer to learn from examples. ü§ñ

  • Statistical Learning: Often used synonymously with Machine Learning, it emphasizes the statistical foundations of learning algorithms. It focuses on building mathematical models to understand and predict data.

Relationship Between These Concepts

  • Data mining often uses machine learning techniques to find patterns.
  • Machine Learning builds on Statistical Learning.
  • All aim to extract insights and make predictions from data.

Introduction to Python and Pandas

  • Python: A versatile, high-level programming language known for its readability and extensive libraries. It‚Äôs a favorite for data analysis and machine learning. üêç

  • Pandas: A powerful Python library specifically designed for data manipulation and analysis. It provides data structures like DataFrame to efficiently store and work with tabular data. Think of it as Excel, but supercharged! üí™

  • Why Pandas?

    • Easy-to-use data structures.
    • Powerful data manipulation tools.
    • Excellent for handling missing data.
    • Built-in functions for data loading, cleaning, and analysis.

Reading and Writing Data in Text Format with Pandas

  • Pandas provides several functions for reading tabular data into a DataFrame object.
  • read_csv is one of the most commonly used functions, designed for reading comma-separated values (CSV) files. But it‚Äôs much more flexible than just CSV!
  • Other functions handle different formats (see the table below).

Common read_csv Optional Arguments

The read_csv function (and others) has many optional arguments, categorized as:

  1. Indexing: Control which columns become the DataFrame‚Äôs index (row labels).
  2. Type Inference and Data Conversion: Pandas tries to automatically detect data types (integer, float, string, etc.). You can customize this.
  3. Date and Time Parsing: Combine and parse date/time information spread across multiple columns.
  4. Iterating: For very large files, read the file in chunks.
  5. Unclean Data Issues: Handle issues like skipping rows, comments, and unusual numeric formats.

Don‚Äôt be overwhelmed by all the options! The pandas documentation has excellent examples. üìö

Example: Reading a Simple CSV

Let‚Äôs start with a basic CSV file (ex1.csv):

a,b,c,d,message
1,2,3,4,hello
5,6,7,8,world
9,10,11,12,foo

We can read this using pd.read_csv:

import pandas as pd
df = pd.read_csv("examples/ex1.csv")
print(df)

The output is:

   a   b   c   d message
0  1   2   3   4   hello
1  5   6   7   8   world
2  9  10  11  12     foo
  • Pandas automatically detected the header row and inferred the data types.
  • The DataFrame is displayed with row indices (0, 1, 2) and column names.

Example: No Header Row

Consider ex2.csv:

1,2,3,4,hello
5,6,7,8,world

Setting the Index

You can specify which column(s) should be the index using index_col:

df = pd.read_csv("examples/ex2.csv", names=names, index_col="message")
print(df)

Handling Non-Standard Delimiters

  • If your file uses a delimiter other than a comma (e.g., spaces, tabs), use the sep argument.

  • For variable whitespace, use a regular expression: sep='\s+'

# Assuming a file 'ex3.txt' with whitespace as a delimiter
result = pd.read_csv("examples/ex3.txt", sep="\s+")
print(result)
         A         B         C
aaa -0.264438 -1.026059 -0.619500
bbb  0.927272  0.302904 -0.032399
ccc -0.264273 -0.386314 -0.217601
ddd -0.871858 -0.348382  1.100491
  • Pandas infers the first column as the index because there‚Äôs one fewer column name than data rows.

Skipping Rows

  • Use skiprows to skip specific rows:
# Assuming a file 'ex4.csv' with comments and a header
df = pd.read_csv("examples/ex4.csv", skiprows=[0, 2, 3])
print(df)
   a   b   c   d message
0  1   2   3   4   hello
1  5   6   7   8   world
2  9  10  11  12     foo

Handling Missing Values

  • Missing data is common in real-world datasets.

  • Pandas uses sentinels like NA and NULL to represent missing values.

  • You can customize these using na_values:

    result = pd.read_csv("examples/ex5.csv", na_values=["NULL"])

  • Disable default NA values with keep_default_na=False.

  • Specify different sentinels per column using a dictionary:

    sentinels = {"message": ["foo", "NA"], "something": ["two"]}
result = pd.read_csv("examples/ex5.csv", na_values=sentinels, keep_default_na=False)

pd.isna()

  • Use pd.isna(dataframe) to check for missing values, which returns a boolean DataFrame.
result = pd.read_csv("examples/ex5.csv")
print(pd.isna(result))
  something      a      b      c      d  message
0     False  False  False  False  False     True
1     False  False  False   True  False    False
2     False  False  False  False  False    False

Reading Text Files in Pieces

  • For very large files, read in chunks using chunksize:
chunker = pd.read_csv("examples/ex6.csv", chunksize=1000)
tot = pd.Series([], dtype='int64')
for piece in chunker:
    tot = tot.add(piece["key"].value_counts(), fill_value=0)

tot = tot.sort_values(ascending=False)
print(tot[:10])
  • The TextFileReader object returned by read_csv with chunksize allows iteration.
  • You can also use the nrows arguments to read small number of rows.

Writing Data to Text Format

  • Use the to_csv method to write a DataFrame to a CSV file:

    data = pd.read_csv("examples/ex5.csv")
data.to_csv("examples/out.csv")

  • Customize the delimiter using sep:

    import sys
data.to_csv(sys.stdout, sep="|") # Output to console for demonstration

Handling Missing Values in Output

  • By default, missing values are represented as empty strings.
  • Use na_rep to specify a different representation:
data.to_csv(sys.stdout, na_rep="NULL")

Controlling Row and Column Labels

  • By default, both row and column labels are written.
  • Disable them using index=False and header=False:
data.to_csv(sys.stdout, index=False, header=False)
  • Write a subset of columns in a specific order:
data.to_csv(sys.stdout, index=False, columns=["a", "b", "c"])

Working with Other Delimited Formats

  • For files with single-character delimiters, use Python‚Äôs built-in csv module:
import csv
f = open("examples/ex7.csv")
reader = csv.reader(f)
for line in reader:
    print(line)
f.close()
  • You can create a dictionary of data columns like this:
with open("examples/ex7.csv") as f:
    lines = list(csv.reader(f))

header, values = lines[0], lines[1:]
data_dict = {h: v for h, v in zip(header, zip(*values))}
print(data_dict)

JSON Data

  • JSON (JavaScript Object Notation) is a common format for web data.
  • It‚Äôs more free-form than CSV.
  • Use the json module to work with JSON data:
import json
obj = """
{"name": "Wes",
 "cities_lived": ["Akron", "Nashville", "New York", "San Francisco"],
 "pet": null,
 "siblings": [{"name": "Scott", "age": 34, "hobbies": ["guitars", "soccer"]},
              {"name": "Katie", "age": 42, "hobbies": ["diving", "art"]}]
}
"""
result = json.loads(obj) # Convert JSON string to Python object
print(result)

asjson = json.dumps(result) # Convert Python object back to JSON
print(asjson)

JSON and Pandas

  • Convert a list of dictionaries (from JSON) to a DataFrame:

    siblings = pd.DataFrame(result["siblings"], columns=["name", "age"])
print(siblings)

  • pd.read_json can automatically convert JSON datasets into a Series or DataFrame:

    data = pd.read_json("examples/example.json")
print(data)

  • Export data to JSON using to_json: python print(data.to_json()) print(data.to_json(orient="records"))

XML and HTML: Web Scraping

  • Python libraries like lxml, Beautiful Soup, and html5lib are used for parsing HTML and XML.
  • pandas.read_html uses these libraries to parse tables from HTML files:
tables = pd.read_html("examples/fdic_failed_bank_list.html")
failures = tables[0]
print(failures.head())
  • You can perform data cleaning and analysis, for example, calculating bank failures by year: python close_timestamps = pd.to_datetime(failures['Closing Date']) print(close_timestamps.dt.year.value_counts())

Parsing XML with lxml.objectify

  • For more complex, structured XML data use lxml.objectify
from lxml import objectify

path = "datasets/mta_perf/Performance_MNR.xml"
with open(path) as f:
    parsed = objectify.parse(f)

root = parsed.getroot()

data = []
skip_fields = ["PARENT_SEQ", "INDICATOR_SEQ", "DESIRED_CHANGE", "DECIMAL_PLACES"]

for elt in root.INDICATOR:
  el_data = {}
  for child in elt.getchildren():
    if child.tag in skip_fields:
        continue
    el_data[child.tag] = child.pyval
  data.append(el_data)

perf = pd.DataFrame(data)
print(perf.head())
  • pandas.read_xml provides a one-line solution for this example: python perf2 = pd.read_xml(path) print(perf2.head())

Binary Data Formats

  • Pickle: Python‚Äôs built-in serialization module. Use to_pickle and read_pickle for short-term storage. python frame = pd.read_csv("examples/ex1.csv") frame.to_pickle("examples/frame_pickle") pd.read_pickle("examples/frame_pickle")

  • Caution: Pickle is not recommended for long-term storage due to potential compatibility issues. ‚ö†Ô∏è

  • Other binary formats supported by pandas: HDF5, ORC, Parquet, etc. These offer advantages like compression and efficient I/O, especially for large datasets.

Reading Microsoft Excel Files

  • Use pandas.ExcelFile or pandas.read_excel:
# Using ExcelFile (efficient for multiple sheets)
xlsx = pd.ExcelFile("examples/ex1.xlsx")
df = xlsx.parse(sheet_name="Sheet1", index_col=0) #read first sheet

  • Write pandas data to excel:
writer = pd.ExcelWriter("examples/ex2.xlsx")
frame.to_excel(writer, "Sheet1")
writer.save()
  • Or simply: frame.to_excel("examples/ex2.xlsx")

Using HDF5 Format

  • HDF5 (Hierarchical Data Format) is designed for storing large, scientific array data.

  • Install PyTables: conda install pytables or pip install tables

frame = pd.DataFrame({"a": np.random.standard_normal(100)})
store = pd.HDFStore("examples/mydata.h5")  # Create HDFStore object
store["obj1"] = frame         # Store DataFrame like a dictionary
store["obj1_col"] = frame["a"] # Store a Series
print(store["obj1"]) #retrieve stored objects
  • HDFStore supports two storage schemas: fixed and table.

-table schema support query operation, like:

store.put("obj2", frame, format="table")
print(store.select("obj2", where=["index >= 10 and index <= 15"]))
store.close()

  • pandas.read_hdf provides a shortcut.

  • HDF5 is not a database. It‚Äôs best for write-once, read-many datasets.

Interacting with Web APIs

  • Many websites offer data through public APIs, often returning JSON.

  • Use the requests library to access these APIs:

    import requests
url = "https://api.github.com/repos/pandas-dev/pandas/issues"
resp = requests.get(url)
resp.raise_for_status() # Check for HTTP errors
data = resp.json()      # Parse JSON response

  • Create a DataFrame from the API response: python issues = pd.DataFrame(data, columns=["number", "title", "labels", "state"]) print(issues)

Interacting with Databases

  • SQL-based relational databases are common in business settings.
  • Use sqlite3 (built-in) for a simple example:
import sqlite3
query = """
CREATE TABLE test
(a VARCHAR(20), b VARCHAR(20),
 c REAL,        d INTEGER
);"""
con = sqlite3.connect("mydata.sqlite")
con.execute(query)
con.commit()

data = [("Atlanta", "Georgia", 1.25, 6),
        ("Tallahassee", "Florida", 2.6, 3),
        ("Sacramento", "California", 1.7, 5)]
stmt = "INSERT INTO test VALUES(?, ?, ?, ?)"
con.executemany(stmt, data)
con.commit()

Reading Data from Database

  • Retrieve data and create a DataFrame:

    cursor = con.execute("SELECT * FROM test")
rows = cursor.fetchall()
df = pd.DataFrame(rows, columns=[x[0] for x in cursor.description])
print(df)

  • SQLAlchemy provides a higher-level interface, abstracting differences between SQL databases. Install using conda install sqlalchemy. python import sqlalchemy as sqla db = sqla.create_engine("sqlite:///mydata.sqlite") df_sql = pd.read_sql("SELECT * FROM test", db) print(df_sql)
