Magic Commands: IPython‚Äôs Superpowers üßô‚Äç‚ôÇÔ∏è

  • Magic commands are special commands unique to IPython, not built-in Python.
  • Prefixed with the percent symbol (%).
  • They streamline common tasks and allow you to control IPython‚Äôs behavior. Think of them as ‚Äúshortcuts‚Äù for IPython itself!

Example: %timeit ‚è±Ô∏è

  • %timeit measures the execution time of a Python statement. Very useful for performance analysis!
import numpy as np
a = np.random.standard_normal((100, 100))
%timeit np.dot(a, a)
  • The output shows the average execution time and standard deviation over multiple runs.
  • In the result 92.5 ¬µs ¬± 3.43 ¬µs per loop (mean ¬± std. dev. of 7 runs, 10000 loops each), ¬µs means microsecond.

Exploring Magic Commands ‚ùì

  • Use ? after a magic command to view its ‚Äúcommand-line‚Äù options (like a help manual).
%debug?

This will show the detailed usage of the %debug command, including how to activate the interactive debugger, set breakpoints, and use it in post-mortem mode.

automagic: Magic Without the % ‚ú®

  • automagic lets you use magic commands without the % prefix, as long as no variable has the same name.
  • Toggle it on/off with %automagic.
%automagic  # Turns automagic on or off
pwd         # Equivalent to %pwd (if automagic is on and 'pwd' isn't a variable)

Assigning Magic Command Output

  • Some magic commands behave like functions; you can assign their output to variables.
directory = %pwd  # Stores the current working directory in 'directory'
print(directory)

Essential Magic Commands üìú

Here are some frequently used IPython magic commands:

Command Description
%quickref Display the IPython Quick Reference Card
%magic Display detailed documentation for all available magic commands
%debug Enter the interactive debugger at the last exception traceback
%hist Print command input history (with optional output)
%pdb Automatically enter debugger after any exception
%paste Execute preformatted Python code from clipboard
%cpaste Open a special prompt for manually pasting code

%page OBJECT Pretty-print an object and display it through a pager

  • Press q to exit the %quickref or %magic pager.

The %run Command üèÉ

  • Executes a Python script within your IPython session.
  • The script runs in an empty namespace (no prior imports or variables).
  • Variables, functions, and imports from the script become accessible in your IPython shell after execution.
# In a file named 'my_script.py':
def my_function(x):
    return x * 2

result = my_function(5)

# In IPython:
%run my_script.py
print(result)  # Output: 10

%run -i: Accessing Existing Variables

  • %run -i lets the script access variables already defined in your IPython session.
# In IPython:
my_var = 10

# In 'script_with_access.py':
print(my_var * 3)

# Back in IPython:
%run -i script_with_access.py  # Output: 30

%load: Importing Scripts into Cells (Jupyter)

  • In Jupyter notebooks, %load imports a script directly into a code cell.
# %load my_script.py  # This will be replaced by the script's content
  • Then, the code cell content will be:
def my_function(x):
    return x * 2

result = my_function(5)

Interrupting Running Code ‚úã

  • Press Ctrl-C to interrupt running code (whether from %run or a long command).
  • Raises a KeyboardInterrupt, stopping most Python programs immediately.
  • Sometimes, with compiled extensions, Ctrl-C might not work instantly.

If Ctrl-C doesn‚Äôt immediately stop execution (especially when compiled extensions are involved), you may need to forcibly terminate the Python process using your operating system‚Äôs tools (e.g., Task Manager on Windows, kill on Linux).

Executing Code from the Clipboard üìã

  • %paste and %cpaste are handy for running code copied from elsewhere (but not needed in Jupyter, where direct pasting works).
  • %paste executes the clipboard content as a single block.
  • %cpaste opens a special prompt, letting you paste multiple blocks and edit before execution. Press Ctrl-D in the prompt to execute, and press Ctrl-C to exit the prompt.
# Example code on clipboard:
x = 5
if x > 2:
    print("x is greater than 2")

# In IPython:
%paste  # Executes the code immediately

Command History: Your IPython Memory üß†

  • IPython keeps a small database of your executed commands. This is incredibly useful for:
    • Searching and reusing past commands.
    • Persisting history between sessions.
    • Logging input/output to a file.

Searching and Reusing Commands üîé

  • Up Arrow (Ctrl-P): Search backward in history, matching typed text.
  • Down Arrow (Ctrl-N): Search forward in history.
  • Ctrl-R: Reverse-i-search (like in bash). Type characters, and it finds matching commands. Press Ctrl-R multiple times to cycle.

Input and Output Variables ‚Ü©Ô∏éÔ∏è

  • IPython stores input and output:
    • _ (one underscore): Previous output.
    • __ (two underscores): Second-to-last output.
    • _iX: Input of line X (e.g., _i5 for input on line 5) - stored as a string.
    • _X: Output of line X.
2 + 2      # Output: 4
result = _  # result is now 4
print(_i1)  # Prints the string "2 + 2"

Working with History: %hist, %reset, %xdel

  • %hist: Prints input history (with or without line numbers).
  • %reset: Clears the interactive namespace (and optionally input/output caches).
  • %xdel variable: Removes a variable and tries to clear IPython‚Äôs internal references. Useful for memory management with large datasets.

IPython‚Äôs input/output history can keep large objects in memory even after you del them. Use %xdel and %reset carefully to avoid memory issues.

Interacting with the Operating System üíª

  • IPython lets you interact with your OS without leaving the shell.
  • Run shell commands, change directories, and capture command output.

Shell Commands with ! üí•

  • Start a line with ! to execute it as a shell command.
!ls  # Lists files in the current directory (Linux/macOS)
!dir # Lists files (Windows)

Capturing Shell Output

  • Assign the output of a ! command to a variable.
file_list = !ls
print(file_list)  # Prints the list of files
  • The returned object is a custom list containing various versions of console output.

Python Variables in Shell Commands: $

  • Use $ to substitute Python variables into shell commands.
my_directory = "data_files"
!ls $my_directory  # Lists files in the 'data_files' directory

Aliases and Bookmarks: Shortcuts for Commands and Directories

  • %alias: Create shortcuts for shell commands.
%alias ll ls -l  # Creates an alias 'll' for 'ls -l'
ll /usr/bin  # Now you can use 'll'
  • %bookmark: Create bookmarks for frequently used directories.
%bookmark mydata /path/to/my/data
cd mydata       # Jumps to the bookmarked directory
%bookmark -l    # Lists all bookmarks
  • Aliases are not persistent between sessions (unless configured). Bookmarks are persistent.

Directory History

  • %cd: Change directory.
  • %pwd: return current directory.
  • %pushd: place current directory on stack and change to target directory.
  • %popd: Change to directory popped off the top of the stack.
  • %dirs: Return a list containing the current directory stack.
  • %dhist: Print the history of visited directories.

Environment Variable

  • %env: Return the system environment variables as a dictionary.
  • %matplotlib: Configure matplotlib integration options

Software Development Tools üõ†Ô∏è

  • IPython isn‚Äôt just for interactive exploration; it‚Äôs also great for software development.
  • Key tools:
    • Interactive Debugger: Enhanced pdb.
    • Code Timing: %time and %timeit.

Interactive Debugger: %debug üêû

  • %debug: Enters the debugger after an exception occurs (post-mortem debugging).
  • Drops you into the stack frame where the error happened.
  • Enhanced with tab completion, syntax highlighting, and traceback context.
# Suppose some_script.py has an error:
%run some_script.py
%debug  # Enters the debugger at the error point

Debugger Commands üß≠

  • Inside the debugger:
    • u (up): Move up the call stack.
    • d (down): Move down the call stack.
    • s (step): Step into a function call.
    • n (next): Execute current line and move to the next line (at the current level).
    • c (continue): Continue execution until the next breakpoint or the end.
    • q (quit): Exit the debugger.
    • p variable: print value of variable
    • !variable : Examine variable contents, when the variable has the same name of a debugger command.

%pdb: Automatic Debugging

  • %pdb: Automatically enters the debugger on any exception. Very useful for debugging!
%pdb on  # Turns on automatic debugging
# Now, any error will drop you into the debugger
%pdb off # Turns it off

Using the Debugger for Development

  • %run -d script.py: Start the debugger before running the script. Type s to step into the script.
  • %run -d -b line_number script.py: Set a breakpoint at line_number.
  • set_trace(): A ‚Äúpoor man‚Äôs breakpoint‚Äù function you can insert into your code.
  • debug(function, *args, **kwargs) : Easily step into a specific function call.

set_trace() and debug() functions

from IPython.core.debugger import Pdb

def set_trace():
    Pdb().set_trace(sys._getframe().f_back)

def debug(f, *args, **kwargs):
    pdb = Pdb()
    return pdb.runcall(f, *args, **kwargs)
  • set_trace() is called within your code to create a temporary breakpoint.
  • debug(f, *args, **kwargs) allows stepping into the function f by passing f along with its positional and keyword arguments.

Code Timing: %time and %timeit (Again)

  • %time: Measures the execution time of a statement once.
  • %timeit: Measures execution time multiple times for a more accurate average. Best for very short operations.
my_list = list(range(100000))
%time for _ in range(100):  sum(my_list)
%timeit sum(my_list)

Basic Profiling: %prun and %run -p üïµÔ∏è‚Äç‚ôÄÔ∏è

  • Profiling tells you where your code spends its time.
  • %prun statement: Profiles a single Python statement.
  • %run -p script.py: Profiles an entire script.
  • Uses cProfile under the hood.
  • Output shows:
    • ncalls: Number of calls.
    • tottime: Total time spent in the function (excluding calls to sub-functions).
    • percall: Time per call (tottime / ncalls).
    • cumtime: Cumulative time spent in the function and its sub-functions.
    • percall: Cumulative time per call (cumtime / ncalls).
    • filename:lineno(function): Location of the function.
#Example
def slow_function():
    result = []
    for i in range(1000):
        result.append(i * 2)
    return result

%prun slow_function()

Line Profiling: %lprun üìè

  • %lprun: Provides line-by-line profiling of specific functions. More detailed than %prun.
  • Requires the line_profiler IPython extension.
    • Enable: add c.InteractiveShellApp.extensions = ['line_profiler'] in ipython_config.py, or use %load_ext line_profiler.
  • Syntax: %lprun -f function1 -f function2 statement_to_profile
  • Output shows:
    • Line #: Line number.
    • Hits: Number of times the line was executed.
    • Time: Total time spent on the line (in timer units).
    • Per Hit: Time per execution of the line.
    • % Time: Percentage of total time spent on the line.
    • Line Contents: The actual code.
# Assuming line_profiler is enabled and prof_mod.py exists:
from prof_mod import add_and_sum, call_function #example file, you need create it.
%lprun -f add_and_sum call_function()

Tips for Productive Code Development üìù

  • Reloading Module Dependencies:
    • Problem: Changes to imported modules aren‚Äôt automatically reflected in IPython.
    • Solutions:
      • importlib.reload(module): Reloads a single module.
      • dreload(module) (IPython-specific): Deep (recursive) reload of a module and its dependencies. More robust.
  • Code Design:
    • Keep relevant objects and data alive: Avoid putting all your code inside a main() function. Keep variables at the top level for easy inspection in IPython.
    • Flat is better than nested: Avoid deeply nested code. Makes debugging and testing easier.
    • Overcome a fear of longer files: Within reason, longer, well-organized modules are often better than many small, interconnected ones (reduces reloading overhead).
    • Maintain high internal cohesion: Related code should be grouped together.

IPython Profiles and Configuration ‚öôÔ∏è

  • Customize IPython‚Äôs appearance and behavior through configuration files (ipython_config.py).
  • Located in your IPython directory (usually ~/.ipython/profile_default/).
  • You can have multiple profiles for different projects or settings.
  • Create a new profile: ipython profile create my_profile
  • Launch with a specific profile: ipython --profile=my_profile
  • Jupyter has its own configuration system.
