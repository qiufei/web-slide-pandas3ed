## Data Mining, Machine Learning, and Statistical Learning

::: {layout-ncol=2}
- Data mining, machine learning, and statistical learning share a common goal: to extract valuable insights and make predictions from data.

- The core of difference is the means to achieve the common goal.

```{mermaid}
graph LR
    A[Data Mining] --> C(Common Ground)
    B[Machine Learning] --> C
    D[Statistical Learning] --> C
    C --> E[Insights & Predictions]

:::

Data Mining

  • Definition: The process of discovering patterns, anomalies, and relationships in large datasets to predict outcomes.

  • Focus: Discovering previously unknown patterns. It’s about exploration and finding something new.

  • Techniques: Often uses a combination of methods from machine learning, statistics, and database systems.

  • Example: A supermarket analyzing purchase data to discover that customers who buy diapers often also buy beer. This is an unexpected association.

Machine Learning

  • Definition: The study of computer algorithms that improve automatically through experience (data). Focuses on prediction and decision making.

  • Focus: Building systems that can learn from and make decisions or predictions based on data, without explicit rules.

  • Types:

    • Supervised Learning: Training a model on a labeled dataset (e.g., classifying emails as spam or not spam).
    • Unsupervised Learning: Discovering hidden structures in unlabeled data (e.g., customer segmentation).
    • Reinforcement Learning: Training agents to make a sequence of decisions by interacting with an environment (e.g., training a robot to walk).

  • Example: Training a model to predict house prices based on features like size, location, and number of bedrooms. The algorithm learns the relationship between features and price.

Statistical Learning

  • Definition: A set of tools for modeling and understanding complex datasets. It’s a framework for applying statistical methods to learn from data.

  • Focus: Emphasis on models and their interpretability, as well as precision and uncertainty. It bridges the gap between statistics and machine learning.

  • Key Concepts:

    • Bias-Variance Tradeoff: Balancing the error introduced by approximating a real-world problem (bias) with the error introduced by the model’s sensitivity to the training data (variance).
    • Model Selection: Choosing the best model from a set of candidate models.
    • Regularization: Techniques to prevent overfitting by adding a penalty for complexity.

  • Example: Using linear regression to understand the relationship between advertising spending and sales, including confidence intervals for the coefficients.

13.1 Bitly Data from 1.USA.gov

  • Background:
    • In 2011, Bitly (URL shortening service) partnered with USA.gov.
    • Provided anonymized data from users shortening links ending in .gov or .mil.
    • Data was available as hourly snapshots (text files).
    • Service is now discontinued, but the data is preserved for analysis.
  • Data Format:
    • Each line in the file is a JSON (JavaScript Object Notation) object.
    • JSON is a human-readable text format for representing data.

  • Key-Value Pairs: Data is stored as key-value pairs (e.g., "a": "Mozilla/...").

Reading JSON Data in Python

import json

path = "datasets/bitly_usagov/example.txt"

with open(path) as f:
    records = [json.loads(line) for line in f]  # List comprehension

print(records[0]) # Print the first record
  • import json: Imports the necessary JSON library.
  • open(path): Opens the file.
  • json.loads(line): Parses a single line of JSON text into a Python dictionary.
  • List Comprehension: [... for line in f] efficiently creates a list of dictionaries, one for each line in the file.
  • records: Now holds a list of Python dictionaries, each representing a record from the dataset.

Accessing Data in the Dictionary

print(records[0]['tz'])  # Accessing the 'tz' (time zone) field of the first record
  • After loading the JSON data, you can access individual fields within each record using standard dictionary key access (square brackets).
  • records[0] gets the first dictionary in the list.
  • ['tz'] accesses the value associated with the key “tz”.

Counting Time Zones (Pure Python)

  • Objective: Determine the most frequent time zones in the dataset (the tz field).
  • Approach 1: Using a Dictionary and Loop
  • Approach 2： Using defaultdict
from collections import defaultdict

def get_counts2(sequence):
    counts = defaultdict(int)  # Values will initialize to 0
    for x in sequence:
        counts[x] += 1
    return counts
  • defaultdict(int): Creates a dictionary where, if a key is not found, it’s automatically added with a default value of 0 (provided by int()). This avoids the if x in counts check.

Counting Time Zones (Pure Python) - continued

  • Key Error Explanation: If the data does not contain the 'tz' key, a KeyError will be raised, as shown on the slide.

def top_counts(count_dict, n=10):

print(frame['a'][1])

Windows vs. Non-Windows Users

import numpy as np

cframe = frame[frame['a'].notna()].copy()  # Create a copy to avoid warnings
cframe['os'] = np.where(cframe['a'].str.contains('Windows'), 'Windows', 'Not Windows')
print(cframe['os'].head())

by_tz_os = cframe.groupby(['tz', 'os'])
agg_counts = by_tz_os.size().unstack().fillna(0) # Group, count, reshape, and fill missing values
print(agg_counts.head())
  • Goal: Analyze time zone data separately for Windows and non-Windows users.
  • cframe = frame[frame['a'].notna()].copy(): Creates a copy of the DataFrame, filtering out rows where the ‘a’ field is missing. Using .copy() avoids a SettingWithCopyWarning that pandas might raise.
  • cframe['a'].str.contains('Windows'): Checks if the agent string contains “Windows” (case-sensitive). Returns a boolean Series.
  • np.where(...): A vectorized way to create a new ‘os’ column based on the condition. If the agent string contains “Windows”, the ‘os’ value is “Windows”; otherwise, it’s “Not Windows”.
  • cframe.groupby(['tz', 'os']): Groups the data by time zone and operating system.
  • .size(): Counts the number of records in each group (similar to value_counts(), but for grouped data).
  • .unstack(): Reshapes the data. It pivots the ‘os’ level from the index to become columns, making it easier to compare Windows and Not Windows counts for each time zone.
  • .fillna(0): Replaces any missing values (which would occur if a time zone had only Windows or only Not Windows users) with 0.

Finding the Most Popular Time Zones

# Method 1: Using argsort
indexer = agg_counts.sum(axis="columns").argsort()
print(indexer.values[:10])
count_subset = agg_counts.take(indexer[-10:])
print(count_subset)

# Method 2: Using nlargest (more convenient)
print(agg_counts.sum(axis="columns").nlargest(10))

  • Goal: Identify the time zones with the highest overall counts (combining Windows and Not Windows).

  • agg_counts.sum(axis="columns"): Calculates the total count for each time zone (summing across the ‘Windows’ and ‘Not Windows’ columns).

  • argsort(): Returns the indices that would sort the array. indexer now holds the indices of time zones, from least frequent to most frequent.

  • .take(indexer[-10:]): Selects the rows corresponding to the top 10 time zones (the last 10 indices in indexer).

  • nlargest(10): A more direct way to get the top 10 time zones. It returns a Series with the time zones and their counts.

Visualizing Windows and Non-Windows Users

# Prepare data for plotting
count_subset = count_subset.stack()
count_subset.name = 'total'
count_subset = count_subset.reset_index()
print(count_subset.head())
sns.barplot(x='total', y='tz', hue='os', data=count_subset)
plt.show()
  • count_subset.stack(): Pivots the ‘os’ column back into the index, creating a multi-level index (tz, os). This is the opposite of unstack(). It’s a common preparation step for seaborn plotting.
  • count_subset.name = 'total': Assigns the name “total” to the Series values (the counts).
  • count_subset.reset_index(): Converts the multi-level index (‘tz’, ‘os’) into regular columns. This makes the data suitable for seaborn’s barplot.
  • sns.barplot(x='total', y='tz', hue='os', data=count_subset): Creates a grouped bar plot.
    • x='total': The ‘total’ column (counts) is used for the x-axis.
    • y='tz': The ‘tz’ column (time zone names) is used for the y-axis.
    • hue='os': The ‘os’ column is used to color the bars (creating separate bars for Windows and Not Windows).
    • data=count_subset: Specifies the DataFrame to use.

Normalizing Counts

def norm_total(group):
    group['normed_total'] = group['total'] / group['total'].sum()
    return group

results = count_subset.groupby('tz').apply(norm_total)
sns.barplot(x='normed_total', y='tz', hue='os', data=results)
plt.show()
  • Goal: Compare the proportion of Windows and non-Windows users within each time zone, rather than the raw counts.
  • norm_total(group) function:
    • Takes a group (DataFrame) as input.
    • Calculates normed_total: the count for each OS within the time zone, divided by the total count for that time zone. This gives the proportion.
    • Returns the modified group.
  • count_subset.groupby('tz').apply(norm_total): Applies the norm_total function to each time zone group.
  • sns.barplot(...): Creates a bar plot, similar to before, but now using ‘normed_total’ to show proportions. This makes it easier to compare the relative prevalence of Windows and non-Windows users in different time zones.

13.2 MovieLens 1M Dataset

  • Dataset: MovieLens 1M, collected by GroupLens Research.
  • Contents:
    • 1 million movie ratings.
    • ~6,000 users.
    • ~4,000 movies.
  • Data Format: Three tables:
    • users: User demographics (age, gender, occupation, zip code).
    • ratings: User ID, movie ID, rating, timestamp.
    • movies: Movie title, genres.
  • Goal: Explore relationships between ratings, demographics, and movie genres.
  • Loading Data:
import pandas as pd
unames = ['user_id', 'gender', 'age', 'occupation', 'zip']
users = pd.read_table('datasets/movielens/users.dat', sep='::',
                      header=None, names=unames, engine='python')
rnames = ['user_id', 'movie_id', 'rating', 'timestamp']
ratings = pd.read_table('datasets/movielens/ratings.dat', sep='::',
                        header=None, names=rnames, engine='python')

mnames = ['movie_id', 'title', 'genres']
movies = pd.read_table('datasets/movielens/movies.dat', sep='::',
                       header=None, names=mnames, engine='python')
  • Key Arguments of pd.read_table:
    • sep='::': The separator between fields is two colons, and the default separator is a comma.
    • header=None: The data files don’t have header rows, so we specify None.
    • names=unames (etc.): We provide the column names.
    • engine='python': Use the python parsing engine, because the default C parsing engine does not support multi-character separators.

Merging the Data

data = pd.merge(pd.merge(ratings, users), movies)
print(data.head())
print(data.iloc[0])  # Access the first row using iloc
  • pd.merge(): Combines DataFrames based on common columns. pandas automatically finds the common columns (e.g., ‘user_id’, ‘movie_id’).
  • Two Merges:
    1. pd.merge(ratings, users): Merges ratings and users based on user_id.
    2. pd.merge(..., movies): Merges the result with movies based on movie_id.
  • data: Now contains all the information in a single DataFrame.
  • data.iloc[0]: show the first record in the combined data.

Analyzing Ratings by Gender

mean_ratings = data.pivot_table('rating', index='title', columns='gender', aggfunc='mean')
print(mean_ratings.head())
  • data.pivot_table(...): A powerful way to reshape and aggregate data.
    • 'rating': The values to aggregate (we want the average rating).
    • index='title': The movie title will be the row index.
    • columns='gender': Gender (‘M’ and ‘F’) will be the columns.
    • aggfunc='mean' (default): Calculates the mean rating for each title/gender combination.

Filtering by Number of Ratings

ratings_by_title = data.groupby('title').size()
print(ratings_by_title.head())
active_titles = ratings_by_title.index[ratings_by_title >= 250]
print(active_titles)

mean_ratings = mean_ratings.loc[active_titles]
print(mean_ratings)

  • data.groupby('title').size(): Counts the number of ratings for each movie title. .size() is used for grouped data (similar to value_counts() for Series).
  • active_titles = ...: Creates a list of movie titles that have at least 250 ratings (an arbitrary threshold).
  • mean_ratings.loc[active_titles]: Filters the mean_ratings DataFrame to include only the movies in active_titles. This focuses the analysis on movies with more substantial rating data.

Sorting and Finding Top Films

top_female_ratings = mean_ratings.sort_values("F", ascending=False)
print(top_female_ratings.head())
  • mean_ratings.sort_values("F", ascending=False): Sorts the mean_ratings DataFrame by the ‘F’ (female) column in descending order. This puts the movies with the highest average female ratings at the top.

Measuring Rating Disagreement

mean_ratings['diff'] = mean_ratings['M'] - mean_ratings['F']

Handling Movie Genres

# Original genres format: "Animation|Children's|Comedy"
movies['genre'] = movies.pop('genres').str.split('|')
print(movies.head())
movies_exploded = movies.explode('genre')
print(movies_exploded[:10])

  • movies['genres'].str.split('|'): Splits the ‘genres’ string by the ‘|’ character, creating a list of genres for each movie.
  • .pop('genres'): Removes the original ‘genres’ column and returns it. This is done to replace the original column with the new list-based one.
  • movies.explode('genre'): Transforms the DataFrame so that each movie-genre combination gets its own row. This is crucial for analyzing ratings by genre. For example, if a movie is both “Action” and “Comedy”, it will now have two rows in movies_exploded.

Combining All Data and Grouping by Genre

ratings_with_genre = pd.merge(pd.merge(movies_exploded, ratings), users)
print(ratings_with_genre.iloc[0])

genre_ratings = (ratings_with_genre.groupby(['genre', 'age'])
                 ['rating'].mean()
                 .unstack('age'))
print(genre_ratings[:10])
  • ratings_with_genre = ...: Merges all three tables (movies_exploded, ratings, and users) to create a single DataFrame where each row represents a single rating, along with user demographics and the individual genre.
  • ratings_with_genre.iloc[0]: Displays the first row of this combined DataFrame.
  • genre_ratings = ...: Calculates the average rating for each genre/age group combination.
    • groupby(['genre', 'age']): Groups by genre and age.
    • ['rating'].mean(): Calculates the mean rating for each group.
    • .unstack('age'): Pivots the ‘age’ level from the index to become columns, making it easy to compare ratings across age groups for each genre.

13.3 US Baby Names 1880–2010

  • Dataset: US Social Security Administration (SSA) baby name data.

  • Time Period: 1880-2010.

  • Data Format: One file per year, each containing:

    • Name
    • Sex
    • Number of births

  • Example File (yob1880.txt):

    Mary,F,7065
Anna,F,2604
...

  • Potential Analyses:

    • Name popularity trends over time.
    • Relative rank of names.
    • Changes in naming diversity.
    • Analysis of first/last letters, vowel/consonant usage.

Loading and Combining the Data

import pandas as pd

# Load a single year (for demonstration)
names1880 = pd.read_csv('datasets/babynames/yob1880.txt',
                        names=['name', 'sex', 'births'])
print(names1880)
print(names1880.groupby('sex')['births'].sum())

# Load all years and combine into a single DataFrame
years = range(1880, 2011)
pieces = []
for year in years:
    path = f'datasets/babynames/yob{year}.txt'
    frame = pd.read_csv(path, names=['name', 'sex', 'births'])
    frame['year'] = year  # Add a 'year' column
    pieces.append(frame)

names = pd.concat(pieces, ignore_index=True)
print(names)
  • names1880 = pd.read_csv(...): Loads a single year’s data into a DataFrame. names argument assigns column titles.
  • names1880.groupby('sex')['births'].sum(): Calculates the total births for each sex in 1880.
  • Looping through years:
    • range(1880, 2011): Iterates through the years.
    • f'datasets/babynames/yob{year}.txt': Uses an f-string to construct the file path for each year.
    • frame['year'] = year: Adds a ‘year’ column to each year’s DataFrame.
    • pieces.append(frame): Adds each year’s DataFrame to the pieces list.
  • names = pd.concat(pieces, ignore_index=True): Combines all the DataFrames in the pieces list into a single DataFrame (names).
    • ignore_index=True: Resets the index, so we don’t have duplicate index values from the individual year files.

Aggregating by Year and Sex

total_births = names.pivot_table('births', index='year',
                                 columns='sex', aggfunc=sum)
print(total_births.tail())
total_births.plot(title='Total births by sex and year')
plt.show()
  • names.pivot_table(...): Calculates the total births for each year/sex combination.
    • 'births': The values to aggregate (total births).
    • index='year': The year will be the row index.
    • columns='sex': Sex (‘F’ and ‘M’) will be the columns.
    • aggfunc=sum: Calculates the sum of births for each year/sex combination.
  • total_births.tail(): Displays the last few rows of the result.
  • total_births.plot(...): Creates a line plot showing the trend of total births over time, separated by sex.

Calculating Proportion of Each Name

def add_prop(group):
    group['prop'] = group['births'] / group['births'].sum()
    return group

names = names.groupby(['year', 'sex']).apply(add_prop)
print(names)
  • Goal: Determine the proportion of babies given each name within each year and sex. This normalizes the data, accounting for changes in the total number of births.
  • add_prop(group) function:
    • Takes a group (DataFrame) as input. A group represents a specific year and sex.
    • group['births'] / group['births'].sum(): Calculates the proportion. It divides the births for each name by the total births for that year and sex.
    • Returns the modified group with the new ‘prop’ column.
  • names.groupby(['year', 'sex']).apply(add_prop): Applies the add_prop function to each year/sex group. This adds the ‘prop’ column to the entire names DataFrame.

Verifying the Proportions

print(names.groupby(['year', 'sex'])['prop'].sum())
  • Sanity Check: It’s a good practice to verify that the calculations are correct.
  • names.groupby(['year', 'sex'])['prop'].sum(): Groups by year and sex, and then sums the ‘prop’ column for each group. The result should be 1.0 for each year/sex combination, because the proportions should add up to 100%.

Extracting the Top 1000 Names

def get_top1000(group):
    return group.sort_values('births', ascending=False)[:1000]

grouped = names.groupby(['year', 'sex'])
top1000 = grouped.apply(get_top1000)
top1000 = top1000.reset_index(drop=True)  # Drop the hierarchical index
print(top1000)
  • Goal: For each year and sex, find the 1000 most popular names. This subset is used for further analysis.
  • get_top1000(group) function:
    • Takes a group (DataFrame) as input (a specific year and sex).
    • group.sort_values('births', ascending=False): Sorts the names within the group by the number of births in descending order (most popular first).
